什么是代理(proxy)?

代理模式是常用的java设计模式，他的特征是代理类与委托类有同样的接口，代理类主要负责为委托类预处理消息、过滤消息、把消息转发给委托类，以及事后处理消息等。代理类与委托类之间通常会存在关联关系，一个代理类的对象与一个委托类的对象关联，代理类的对象本身并不真正实现服务，而是通过调用委托类的对象的相关方法，来提供特定的服务。
按照代理的创建时期，代理类可以分为两种。
静态代理：由程序员创建或特定工具自动生成源代码，再对其编译。在程序运行前，代理类的.class文件就已经存在了。
动态代理：在程序运行时，运用反射机制动态创建而成。

​ 由于在日常使用代理功能基本不使用静态代理,本文将从利用java的反射机制的动态代理和cglib实现的动态代理

java动态代理

​ JDK的动态代理依靠接口实现，如果有些类并没有实现接口，则不能使用JDK代理

#### 创建代理对象

首先,我们需要提供一个调用处理器(InvocationHandler),InvocationHandler是实现了(interface InvocationHandler)接口的类对象,在demo中,这个调用处理器是HelloPorxy类.这个接口中有一个invoke方法需要重写. 当我们通过代理对象调用一个方法的时候，这个方法的调用就会被转发为由 InvocationHandler 这个接口的 invoke 方法来进行调用。 

   public Object invoke(Object proxy, Method method, Object[] args)
        throws Throwable;
}

无论何时调用代理对象的方法,调用处理器的invoke方法都会被调用,向其传递Menthod对象和参数,调用处理器必须给出处理调用的方式

要想创建一个代理对象,需要使用Proxy类的newProxyInstance方法,这个方法需要三个参数

public static Object newProxyInstance(ClassLoader loader,                                      Class<?>[] interfaces, InvocationHandler h)
  • 一个类加载器
  • 一个Class对象数组 表示的是我将要给我需要代理的对象提供一组什么接口，如果我提供了一组接口给它，那么这个代理对象就宣称实现了该接口(多态)，这样我就能调用这组接口中的方法了
  • 一个上面我们说到的调用处理器

接下来,我们用例子来说明

例子

这是一个HelloService接口,里面有一个方法,功能是打印名字.

public interface HelloService {

    void printname(String name);
}

实现类实现功能

public class HelloServiceImpl implements HelloService {
    @Override
    public void printname(String name) {
        System.out.println(name);
    }
}

下一步,建立调用处理器

public class HelloPorxy implements InvocationHandler {
    //这个target便是你代理的真实对象.
    private Object target;

    //构造参数,为target传值.
    public HelloPorxy(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {


        before();
        Object o = method.invoke(target, args);//执行完before()后,执行这条语句的时候,会调用invoke动态调用到target真实对象,并调用方法.
        after();
        return o;

    }

    private void before() {
        System.out.println("他来了");
    }

    private void after() {
        System.out.println("他是你永远得不到的男人");
    }

    //利用泛型,简化了代码,否则在使用的时候需要强制类型转换.
    public <T> T getProxy()
    {
         return (T) Proxy.newProxyInstance(target.getClass().getClassLoader(),target.getClass().getInterfaces(),this);

    }


}

主类

public class Main {

    public static void main(String[] args) {
    // write your code here
        HelloService helloService = new HelloServiceImpl();
        HelloPorxy helloPorxy = new HelloPorxy(helloService);
        HelloService service = helloPorxy.getProxy();
        service.printname("程序员");
         //System.out.println(service.getClass().getName()); 要点解析中讲解

    }
}

结果

他来了
程序员
他是你永远得不到的男人

要点解析

第一个疑问

首先我们先看看getProxy()这个方法,

Proxy.newProxyInstance(target.getClass().getClassLoader(),target.getClass().getInterfaces(),this),正如上面所说,newProxyInstance 需要三个参数,target是我们代理的真实对象,第一个参数—使用类加载器来加载我们的真实对象;第二个参数—一个对象数组,里面都是真实对象实现的接口,这样我们就能通过代理类来调用接口中的方法了;第三个参数—-this代表的就是HelloPorxy这个调用处理器本身.

用大白话说就是getProxy()做到了 通过newProxyInstance方法通过创造了一个代理对象,这个代理对象代理的的是target这个真实对象,并且指定了关联使用哪个调用处理器.

​ 第二个疑问

打印System.out.println(service.getClass().getName());这条语句

结果是com.sun.proxy.$Proxy0,为什么不是HelloService类型呢?

为什么呢?原因在于newProxyInstance的第二个参数上,结合例子来说,target.getClass().getInterfaces() 我们为代理类提供了target的实现接口,我们当然可以把这个代理类强制类型转化成接口的类型,demo中强转成了HelloService类型.

​ 第三个疑问

service.printname("程序员");

这个语句做了什么呢?

A: HelloService service = helloPorxy.getProxy(); 这里是通过代理类来调用实现接口中的方法,执行时,它会跳转到代理对象关联的调用处理器上(HelloPorxy).在代码

HelloPorxy helloPorxy = new HelloPorxy(helloService);

这个调用处理器通过接收参数(helloService),生成target真实对象为HelloService的一个调用处理器.进而执行调用处理器的invoke方法.

Cglib动态代理

准备jar包

为什么除了chlib.jar包外还有一个asm.jar包呢?

​ 如果不加上asm.jar包,会报一个错误Exception in thread "main" java.lang.NoClassDefFoundError: org/objectweb/asm/Typ

在谷歌查阅资料得知—- 上网查了以下知道了很多java字节码操作和分析的第三方类库都引用了asm.jar文件，由于工程不是Maven管理的，无法解决以来传递问题，所以要手动引入asm.jar文件。把asm.jar文件添加到项目路径类，运行，然后就正常了。

Cglib的优势

JDK中提供的生成动态代理类的机制有个鲜明的特点是： 某个类必须有实现的接口，而生成的代理类也只能代理某个类接口定义的方法，比如：如果上面例子的HelloServiveimpl实现了HelloService接口的方法,则可以使用JDK的动态代理！极端的情况是：如果某个类没有实现接口，那么这个类就不能同JDK产生动态代理了！ “CGLIB（Code Generation Library），是一个强大的，高性能，高质量的Code生成类库，它可以在运行期扩展Java类与实现Java接口。”

沿用上面的接口类,接口实现类,主类. 调用处理器的功能被cglib代替

例子

实现 MethodInterceptor接口，用来处理 对代理类上所有方法的请求（这个接口和JDK动态代理InvocationHandler的功能和角色是一样的

package me.superning.porxyutils;

import net.sf.cglib.proxy.Enhancer;
import net.sf.cglib.proxy.MethodInterceptor;
import net.sf.cglib.proxy.MethodProxy;

import javax.management.ObjectName;
import java.lang.reflect.Method;

/**
 * @author superning
 */
public class CglibProxy implements MethodInterceptor {

    public <T> T getproxy(Class <T> tClass)
    {
        return (T) Enhancer.create(tClass,tClass.getInterfaces(),this);

    }


    @Override
    public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {

        before();
        Object invoke = methodProxy.invokeSuper(o, objects);
        after();

        return invoke;
    }



    private void before() {
        System.out.println("他来了");
    }

    private void after() {
        System.out.println("他是你永远得不到的男人");
    }
}

主类

package me.superning;

import me.superning.porxyutils.CglibProxy;
import me.superning.service.HelloService;
import me.superning.service.impl.HelloServiceImpl;

/**
 * @author superning
 */
public class Main {

    public static void main(String[] args) {
        CglibProxy cglibProxy = new CglibProxy();

        HelloService service = cglibProxy.getproxy(helloService.getClass());

        //System.out.println(service.getClass().getName()); 在后续的要点解析中讲解
        service.printname("程序员");

    }
}

结果不变

第一个疑问

刨析一下public <T> T getproxy(Class <T> tClass){ return (T) Enhancer.create(tClass,tClass.getInterfaces(),this);}

这个我是用泛型简写优化过的. 和下面的例子结果是一样的.

CglibProxy cglibProxy = new CglibProxy();
        //cglib 中加强器，用来创建动态代理
        Enhancer enhancer = new Enhancer();  
                 //设置要创建动态代理的类,即真实对象
        enhancer.setSuperclass(helloService.getClass());  
               // 设置回调，这里相当于是对于代理类上所有方法的调用，都会调用CallBack，而Callback则需要实行intercept()方法进行拦截
                enhancer.setCallback(cglibProxy);
