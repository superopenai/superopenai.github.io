Java NIO的Buffer组件

经典步骤

使用Buffer时我们通常会有四步

  • 往buffer缓冲区中写数据
  • 使用buffer.filp()进行读写模式转换
  • 从buffer缓冲区里读出数据
  • 使用buffer.clear()来清空缓冲区,抑或是使用buffer.compact()来清除已经读出了的数据.

这是一个小例子

public void printFormFile() {

        try {
            //这个被读取的文件里共有86字节的内容
            RandomAccessFile file = new RandomAccessFile("src/me/superning/file/test.txt", "rw");
            FileChannel channel = file.getChannel();
            //申请一个100字节大小的缓冲区
            ByteBuffer buffer = ByteBuffer.allocate(100);
            //往缓冲区里写入数据
            int read = channel.read(buffer);
            System.out.println("length"+read);


            while (read!=-1)
            {
                //把buffer切换模式 换成读出模式
                buffer.flip();

                while (buffer.hasRemaining())
                {
                    System.out.println("this element:"+(char)buffer.get());
                }
            //让read 变成了-1;
                buffer.clear();

             read =    channel.read(buffer);

                System.out.println(read);

            }
            file.close();

        } catch (FileNotFoundException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }

buffer的三个属性(不变量?)(逃

position和 limit 的意义依赖与buffer的读写模式不同,而Capacity的意思没有什么特别的,就是指容量大小

position

在写入(write)模式下,position将标记着写入的数据,初始值为0,每当读入一个单位的内容,position就会增加一个单位,position最大值是capacity-1.

当使用filp()转换成读(read)模式时,position重置为0,一直标记下一个读出的单位内容.

Limit

在写入(write)模式下,limit的意义和capacity意义相同,表示能读入的最多值.

当使用filp()转换成读(read)模式时,limit将会重置为(write)模式下的position的位置

换句话说，我们可以读取写入的字节数

Buffer 组件类

  • ByteBuffer
  • MappedByteBuffer
  • CharBuffer
  • DoubleBuffer
  • FloatBuffer
  • IntBuffer
  • LongBuffer
  • ShortBuffe

可以看出不同的组件类对应着Java中不同的类型，MappedByteBuffer有些特殊，将会单独介绍。

怎么获得Buffer对象

要获得Buffer对象，你必须要分配一块空间给buffer对象,下面是两个例子

ByteBuffer bytebuffer = new ByteBuffer.allocate(48)
CharBuffer charbuffer = new CharBuffer.allocate(1024)

怎么向Buffer中写数据

  • 使用Channel来向Buffer中写入数据channel.read(buffer)（经常用）
  • 使用put()方法直接向Buffer写入数据
1.
            FileChannel channel = file.getChannel();
            ByteBuffer buffer = ByteBuffer.allocate(100);
            int read = channel.read(buffer);

2.
    buffer.put("hello");

*对于put() 方法，Buffer的不同子类也都给出了相应的实现，使用时参考 JavaDoc *

怎么从Buffer里读数据

  • 使用Channel来从Buffer中读走数据channel.write(buffer) (经常用)
  • 使用Buffer的get() 方法，一个一个取出。
1.
    FileChannel outfileChannel = outfile.getChannel();
    //和写入例子的buffer是同一个buffer
    int write = outfileChannel.write(buffer);

2.
    buffer.get();

演示例子

public void printFormFile() {

        try {
            RandomAccessFile infile = new RandomAccessFile("src/me/superning/file/test.txt", "rw");
            RandomAccessFile outfile = new RandomAccessFile("src/me/superning/file/to.txt", "rw");

            FileChannel channel = infile.getChannel();
            FileChannel outfileChannel = outfile.getChannel();

            //缓冲区分配50byte，比文件中的数据要小。
            ByteBuffer buffer = ByteBuffer.allocate(50);
            int read = channel.read(buffer);
            System.out.println("读入的数据"+read);


            while (read!=-1)
            {
                //转换模式
                buffer.flip();

                int write = outfileChannel.write(buffer);
                System.out.println("写出的数据"+write);

                buffer.clear();
                System.out.println("-----------------------");
                //当文件里的数据都读过一次，让read 变成了-1;
             read =    channel.read(buffer);

                System.out.println("读入的数据"+read);
                if (read==-1)
                {
                    System.out.println("IO结束");
                }

            }
            infile.close();
            outfile.close();

        } catch (FileNotFoundException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }

rewind方法

Buffer.rewind()将position重新设置为0，这样你就可以重新读取缓冲区中的所有数据。在limit保持不变，因此仍标记着可以写入多少数据。

equals() and compareTo()

equal():

两个buffer如果相等，必须符合以下条件：

  • 属于同一种类型（byte，char）
  • 拥有相同数量的元素
  • 所有的元素都相同

compareTo():

compareTo（）方法比较两个缓冲区的其余元素（字节，字符等），以用于例如 排序例程。 在以下情况下，一个缓冲区被认为比另一个缓冲区“小”：

  • 一个缓冲区中相应元素小于另一个缓冲区中的元素。

  • 所有元素都相等，但是第一个缓冲区先于第二个缓冲区用完（所有元素更少），然后用完所有元素。
