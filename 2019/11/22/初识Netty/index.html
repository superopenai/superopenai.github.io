初识Netty

简介

1）本质：JBoss做的一个Jar包

2）目的：快速开发高性能、高可靠性的网络服务器和客户端程序

3）优点：提供异步的、事件驱动的网络应用程序框架和工具

通俗的说：一个好使的处理Socket的框架

安装

安装十分简单，去maven的中央仓库搜索Netty，然后将链接添加至您的项目里的pom.xml文件里即可。不过要注意使用Netty4.x的版本，你可以看到5.x版本已经很久没更新了。

概况

要了解Netty，就必须需要知道Netty的内部总览，Netty有如下几个核心组件需要了解

  • Bootstarp(引导启动类)

  • EventLoopGroup

  • EventLoop

  • SocketChannel

  • Channellnitializer

  • ChannelPipeline

  • ChannelHandler

接下来将各自介绍每个部分

Bootstarp

引导类将会引导启动Netty，包括开启线程，打开套接字等等

EventLoopGroup

可以将多个EventLoop组合在一起。通过这种方式，EventLoop可以共享线程等资源。

EventLoopGroup

EventLoop是一个循环，不断寻找新事件，例如 来自网络套接字（来自SocketChannel）实例的传入数据。 发生事件时，该事件将传递到适当的事件处理程序，例如ChannelHandler。

SocketChannel

SocketChannel代表着与其他网络计算机的Tcp连接，无论使用Netty作为客户端还是服务器端，所有与其他计算机进行的数据交换，都要通过一个SocketChannel实例，这个实例代表的是和其他计算机之间的连接。

一个SocketChannel由一个EvnetLoop来管，而且只由这个EventLoop来管理。EventLoop总是由同一个线程执行，而SocketChannel实例也只由同一个线程访问。 在使用 SocketChannel不用担心线程同步问题

ChannelInitializer

Netty ChannelInitializer是一个特殊的ChannelHandler，在创建SocketChannel时，它将附加到SocketChannel的ChannelPipeline。 然后调用ChannelInitializer，以便它可以初始化SocketChannel。

初始化SocketChannel后，ChannelInitializer会将其自身从ChannelPipeline中删除。

ChannelPipeline

每个 SocketChannel都有一个ChannelPipeline。 ChannelPipeline包含ChannelHandler实例列表。 当EventLoop从SocketChannel读取数据时，数据将传递到ChannelPipeline中的第一个ChannelHandler。 第一个ChannelHandler可以处理数据，可以选择将其转发到ChannelPipeline中的下一个ChannelHandler，然后，下一个ChannelHandler 可以选择处理程序也可以选择将其转发至ChannelPipeline中的下一个ChannelHandler。

当将数据写到SocketChannel时，在最终写入SocketChannel之前，写入的数据还将通过ChannelPipeline传递。

ChannelPipeline 是Netty里非常核心的一个概念，每一个SocketChannel都包含着一个ChannelPipeline ，每一个ChannelPipeline 里则包含着由许多ChannelHandler构成的列表，这些处理器将会在SocketChannel写入或者读出数据时，被调用。

  • ChannelInboundHandler
  • ChannelOutboundHandler

这两个子处理器分别对应着写入和读出。

从SocketChannel接收到数据后，该数据将传递到ChannelPipeline中的第一个ChannelInboundHandler。 此ChannelInboundHandler处理数据，然后将数据传递到ChannelPipeline中的下一个ChannelInboundHandler。实际上，ChannelInboundHandler可以转换接收到的数据，然后再将其传递到管道中的下一个处理程序。 例如，原始字节可以转换为HTTP对象或其他一些对象。 然后，管道中的下一个处理程序将看到HTTP对象，而不是原始数据。

当将数据写回到SocketChannel时，也会相同的方式发生。 数据从ChannelOutboundHandler传递到ChannelPipeline中的ChannelOutboundHandler，直到到达SocketChannel。 ChannelOutboundHandler实例还可以转换流程中的数据。

尽管例图将ChannelInboundHandler和ChannelOutboundHandler实例显示为单独的列表，但它们实际上位于同一列表（管道）中。 因此，如果ChannelInboundHandler决定将某些内容写回SocketChannel，则数据将通过比当时使用的那个ChannelInboundHandler写入数据更早的那些ChannelOutboundHandler实例把数据处理再读出到SocketChannel。

ChannelHandler

ChannelHandler处理从Netty SocketChannel接收到的数据。 ChannelHandler还可以处理正在写出到SocketChannel的数据。

代码示例（服务端）


/**
 * @author superning
 */

@Service
public class HelloServerImpl implements HelloServer {
    @Override
    public void helloServer() throws InterruptedException {



        EventLoopGroup group = new NioEventLoopGroup();

        ServerBootstrap bootstrap = new ServerBootstrap();
        //连结相应的EventLoopGroup用于处理要创建的通道的所有事件
        bootstrap.group(group);

        //将会创造和 该MioserverSocketChannel相同的通道实例
        bootstrap.channel(NioServerSocketChannel.class);
        //设置本地地址和端口号
        bootstrap.localAddress(newInetSocketAddress("localhost",44444));

        /*
        创建ChannelInitializer并将其附加到ServerBootstrap实例。 ChannelInitializer初始化所有传入TCP连接的套接字。在以下行中创建并附加ChannelInitializer
        */

        bootstrap.childHandler( new ChannelInitializer<SocketChannel>() {
            /*
             ChannelInitializer类是一个抽象类。 每当TCP服务器接受新的传入TCP连接时，就会调用其方法initChannel（）。 它将新的HelloServerHandler（一个ChannelHandler）附加到每个新的SocketChannel。 还可以在所有新的SocketChannel实例中重用相同的ChannelHandler，而不是每次都在此处创建新的实例。
            */
                    @Override
                    protected void initChannel(SocketChannel socketChannel) throws Exception {
                        socketChannel.pipeline().addLast(new HelloServerHandler());          
                    }});


        try {
            //开启服务器
            ChannelFuture sync = bootstrap.bind().sync();
            System.out.println("Server start listen at " );

            sync.channel().closeFuture().sync();

        }
        catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            group.shutdownGracefully().sync();

        }



    }
}
/**
 * @author superning
 */
public class HelloServerHandler extends ChannelInboundHandlerAdapter {

    @Override
    //重写方法,写我们自己的逻辑
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {

        ByteBuf inbuffer = (ByteBuf) msg;
        String s = inbuffer.toString(CharsetUtil.UTF_8);
        System.out.println("server recive "+ s);
        ctx.write((Unpooled.copiedBuffer("Hello " + s, CharsetUtil.UTF_8)));
    }

    @Override
    public void channelReadComplete(ChannelHandlerContext ctx) throws Exception {
        ctx.writeAndFlush(Unpooled.EMPTY_BUFFER).addListener(ChannelFutureListener.CLOSE);

    }

    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {

            cause.printStackTrace();
            ctx.close();




    }

要创建Netty TCP服务器，必须：

  • 创建一个EventLoopGroup
  • 创建和配置ServerBootstrap
  • 创建一个ChannelInitializer
  • 启动服务器

代码示例 (客户端)

/**
 * @author superning
 */
@Service
public class HelloClientImpl implements HelloClient {
    @Override
    public void helloClient() throws InterruptedException {
        NioEventLoopGroup group = new NioEventLoopGroup();
        Bootstrap clientBootstrap = new Bootstrap();
        clientBootstrap.group(group);
        clientBootstrap.channel(NioSocketChannel.class);
//        clientBootstrap.remoteAddress(new InetSocketAddress(9999));
        clientBootstrap.handler(new ChannelInitializer<SocketChannel>() {
            @Override
            protected void initChannel(SocketChannel socketChannel) throws Exception {
                socketChannel.pipeline().addLast(new HelloClientHandler());
            }
        });
        try {
            ChannelFuture sync = clientBootstrap.connect(new InetSocketAddress("127.0.0.1",9999)).sync();
            sync.channel().closeFuture().sync();

        } catch (InterruptedException e) {
            e.printStackTrace();
        }finally {
            group.shutdownGracefully().sync();

        }


    }
/**
 * @author superning
 */
public class HelloClientHandler extends SimpleChannelInboundHandler {

    @Override
    public void channelActive(ChannelHandlerContext channelHandlerContext) {
        channelHandlerContext.writeAndFlush(Unpooled.copiedBuffer("Netty 刘希宁", CharsetUtil.UTF_8));
    }


    @Override
    public void exceptionCaught(ChannelHandlerContext channelHandlerContext, Throwable cause) {
        cause.printStackTrace();
        channelHandlerContext.close();
    }

    @Override
    protected void channelRead0(ChannelHandlerContext channelHandlerContext, Object o) throws Exception {
        ByteBuf in = (ByteBuf) o;


        System.out.println("Client received: " +in.toString(CharsetUtil.UTF_8));

    }

#### 注意事项

因为上面的例子中都使用的本地地址127.0.0.1,所以看不出来一个问题,即服务端填写IP地址的时候注意使用 本地地址,对应到真正的远程部署的时候,就需要填写你的服务器的内网地址,客户端填写的IP地址就是服务器的公网地址.
