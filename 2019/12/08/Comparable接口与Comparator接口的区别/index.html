Comparable接口与Comparator接口的区别

Comparable接口

通过阅读翻译openjdk 源码关于Comparable接口的解释描述可以得知：

Comparable接口会对所有实现它的对象施加一个总排序，这个总排序被称为类的自然排序，CompareTo方法称为类的自然比较方法。

实现这个接口的列表和数组会通过Collections.sort/Arrays.sort实现自动排序

当且仅当e1.compareTo（e2）== 0对于类的每个e1和e2具有与e1.equals（e2）相同的布尔值时，才可以认为C类的自然排序与equals一致

这个接口有什么用呢？—–> 例如一个数组实现了这个接口，那么它排序就十分简单（ Arrays.sort()）,如果一个List实现了这个接口，那么就使用Collections.sort()来给其排序。事实上几乎所有的java等值类都实现了这个接口，这个接口也只有一个方法。

public interface Comparable<T> {
    public int compareTo(T o);
}
  • this < obj 返回负**
  • this = obj 返回 0
  • *this > obj 返回正 *

示例代码

package me.superning.nettychat.Initialzer;

import java.util.ArrayList;
import java.util.Arrays;
import java.util.Collections;
import java.util.List;

public class demo {


    public static void main(String[] args) {
        List<person> people = new ArrayList<person>();
        people.add(new person("a",10));
        people.add(new person("b",5));
        people.add(new person("c",80));
        people.forEach(person -> {
            System.out.println(person.name+"-----"+person.age);
        });

        //使用了显式的比较器
        Collections.sort(people);
        people.forEach(person -> {
            System.out.println(person.name+"-----"+person.age);
        });

    }





}
class person implements Comparable<person> {
    public person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String name;
    public int age;


    @Override
    public int compareTo(person o) {
        return this.age-o.age;

    }
}



结果：
a-----10
b-----5
c-----80

b-----5
a-----10
c-----80

抛出的异常

  • NullPointException: 入参为null

  • ClassCastException: 入参类型与被比较类型不兼容，如执行new Integer(1).compareTo(new Double(2.1));时，编译错误：Error:(35, 23) java: 不兼容的类型: java.lang.Double无法转换为java.lang.Integer

Comparator接口

说到现在，既然有了一个Comparable接口，为什么还要有一个Comparator接口呢？

我们需要了解一下策略模式：

​ 策略模式定义了一系列的算法，并将每一个算法封装起来，而且使它们还可以相互替换。策略模式让算法独立于使用它的客户而独立变化。（原文：The Strategy Pattern defines a family of algorithms,encapsulates each one,and makes them interchangeable. Strategy lets the algorithm vary independently from clients that use it.）

继续使用Person类来说说该如何通过使用Comparator接口来帮助我们实现排序
Person类使用的是没有实现任何接口的Person类。
那么我们该如何对Person类的对象进行排序呢？——-使用Comparator的实现类来帮助我们排序
