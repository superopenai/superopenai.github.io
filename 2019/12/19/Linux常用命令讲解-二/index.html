iostat命令

iostat 用于监控统计系统输入输出设备和 CPU 的使用情况，英文 I/O statistics 的缩写。

它的特点是汇报磁盘活动统计情况，同时也会汇报出 CPU 使用情况。同 vmstat 一样，iostat 也有一个弱点，就是它不能对某个进程进行深入分析，仅对系统的整体情况进行分析。iostat 属于 sysstat 软件包，所以使用前需先安装 sysstat。

root@iZj6c8ezzawm4vybolwi68Z:~# iostat -help
Usage: iostat [ options ] [  [  ] ]
Options are:
[ -c ] [ -d ] [ -h ] [ -k | -m ] [ -N ] [ -t ] [ -V ] [ -x ] [ -y ] [ -z ]
[ -j { ID | LABEL | PATH | UUID | ... } ]
[ [ -H ] -g  ] [ -p [  [,...] | ALL ] ]
[  [...] | ALL ]


可从上面看出， iostat的命令格式---iostat [参数] [间隔时间] [报告次数]

常用参数

  • -C 显示 CPU 使用情况

  • -d 显示磁盘使用情况

  • -k 以 KB 为单位显示

  • -m 以 M 为单位显示

  • -N 显示磁盘阵列(LVM) 信息

  • -n 显示 NFS 使用情况

  • -p[磁盘] 显示磁盘和分区的情况

  • -t 显示终端和 CPU 的信息

  • -x 显示详细信息

  • -V 显示版本信息

    更多参数和详细说明 可以参考 说明文档 iostat - Linux man page

属性说明

CPU 属性值

标示 说明

%idle CPU 空闲时间百分比

标示 说明

%util 被 I/O 需求消耗的 CPU 百分比

TPS 属性值

标识 说明

kB_read 读取的总数据量

iotop命令

iotop命令是一个用来监视磁盘I/O使用状况的top类工具。iotop具有与top相似的UI，其中包括PID、用户、I/O、进程等相关信息。Linux下的IO统计工具如iostat，nmon等大多数是只能统计到per设备的读写情况，如果你想知道每个进程是如何使用IO的就比较麻烦，使用iotop命令可以很方便的查看。

iotop使用Python语言编写而成，要求Python2.5（及以上版本）和Linux kernel2.6.20（及以上版本）。iotop提供有源代码及rpm包，可从其官方主页下载。

常用参数

  • --version 显示程序的版本号并退出
  • -h， –help显示此帮助消息并退出
  • -o， -仅显示实际正在执行I / O的进程或线程
  • -b， –batch非交互模式
  • -n NUM， –iter = NUM结束之前的迭代次数[infinite]
  • -d SEC， –delay =迭代之间的SEC延迟[1秒]
  • -p PID，-pid = PID进程/线程以监视[全部]
  • -u USER，–user = USER用户来监视[全部]
  • -P，–processes仅显示进程，而不是所有线程
  • -a，–accumulated显示累积的I / O而不是带宽
  • -k，-千字节使用千字节而不是人类友好的单位
  • -t，–time在每行上添加一个时间戳（暗示–batch）
  • -q，–quiet禁止显示标题的某些行（暗示–batch）

blktrace命令

blktrace是一个针对Linux内核中块设备I/O层的跟踪工具，是由Linux内核块设备层的维护者开发的，目前已经集成到内核2.6.17及其之后的内核版本中。通过使用这个工具，使用者可以获取I/O请求队列的各种详细的情况，包括进行读写的进程名称、进程号、执行时间、读写的物理块号、块大小等等，是一个Linux下分析I/O相关内容的很好的工具，在使用时基本上只会占用系统2%的资源 。

使用示例

blktrace区分两种类型的块层请求，文件系统和SCSI命令。前者被称为fs请求，后者被称为pc请求。文件系统请求是正常的读/写操作，即从给定大小的特定磁盘位置进行任何类型的读或写。这些请求通常源自用户进程，但也可能是由虚拟机将脏数据刷新到磁盘或文件系统将超级块或日志块同步到磁盘引发的。pc请求是SCSI命令。blktrace将命令数据块作为有效载荷发送，以便blkparse可以对其进行解码。

要跟踪设备/dev/hda上的i/o并将输出解析为可读形式，请使用以下命令:

% blk trace-d/dev/sda-o-| blk rse-I-

要跟踪设备上的i/o并保存输出供以后使用blktrace处理，请使用如下命令

% blktrace /dev/sda /dev/sdb

这将跟踪设备/dev/sda和/dev/sdb上的i/o，并将记录的信息分别保存在两个不同设备的当前目录中的sda和sdb文件中。blkparse实用程序稍后可以解析该跟踪信息:

% blkparse sda sdb

它将以友好的显示形式将先前记录的跟踪信息输出。

swapon命令

swapon命令用于激活Linux系统中交换空间，Linux系统的内存管理必须使用交换区来建立虚拟内存。

常用参数

swapon [选项] [<spec>]

启用设备和文件以进行分页和交换。

选项：

  • -a，–all启用/ etc / fstab中的所有交换

  • -d，–discard [= ]启用交换丢弃（如果设备支持）

  • -e，–ifexists静默跳过不存在的设备

  • -f，–fixpgsz必要时重新初始化交换空间

  • -o，–options <列表>逗号分隔的交换选项列表

  • -p，–priority 指定交换设备的优先级

  • -s，-summary显示有关已用交换设备的摘要（不建议使用）

    –show [= <列>]在可定义表中显示摘要

    –noheadings不打印表格标题（使用–show）

    –raw使用原始输出格式（带有–show）

    –bytes在–show输出中以字节为单位显示交换大小

  • -v，–verbose详细模式

  • -h，–help显示此帮助并退出

  • -V，–version输出版本信息并退出

参数：

  • -L
  • -U UUID = 的同义词
  • LABEL =
  • UUID = 通过交换区UUID指定设备
  • PARTLABEL =
  • PARTUUID = 通过分区UUID指定设备
  • 要使用的设备名称
  • 要使用的文件名

可用的丢弃策略类型（用于–discard）：

  • once：仅发布一次性区域丢弃物
  • pages：释放的页面在重用之前将被丢弃

如果未选择任何策略，则将同时启用两种丢弃类型（默认）。

可用列（用于–show）：

  • NAME 设备文件或分区路径
  • TYPE 设备的类型
  • SIZE 交换区域的SIZE大小
  • USED 使用中已使用字节
  • PRIO 交换优先级
  • UUID 交换uuid
  • LABEL 交换标签

使用示例

# swapon -v /dev/hdb4
# swapon -s
Filename                                type            Size    Used    Priority
/dev/hda5                               partition       506008 96      -1
/dev/hdb4                               partition       489972 0       -2

从整体上看，netstat的输出结果可以分为两个部分：

一个是Active Internet connections，称为有源TCP连接，其中”Recv-Q”和”Send-Q”指%0A的是接收队列和发送队列。这些数字一般都应该是0。如果不是则表示软件包正在队列中堆积。这种情况只能在非常少的情况见到。

另一个是Active UNIX domain sockets，称为有源Unix域套接口(和网络套接字一样，但是只能用于本机通信，性能可以提高一倍)。
Proto显示连接使用的协议,RefCnt表示连接到本套接口上的进程号,Types显示套接口的类型,State显示套接口当前的状态,Path表示连接到套接口的其它进程使用的路径名。

常用参数

  • -a (all)显示所有选项，默认不显示LISTEN相关
  • -t (tcp)仅显示tcp相关选项
  • -u (udp)仅显示udp相关选项
  • -n 拒绝显示别名，能显示数字的全部转化成数字。
  • -l 仅列出有在 Listen (监听) 的服務状态
  • -p 显示建立相关链接的程序名
  • -r 显示路由信息，路由表
  • -e 显示扩展信息，例如uid等
  • -s 按各个协议进行统计
  • -c 每隔一个固定时间，执行该netstat命令。
