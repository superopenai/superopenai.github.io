深化理解netty中的常用组件（个人向)

Inbound和Outbound

开始的错误理解

ChannelInboundHandler , ChanneloutboundHandler 当初第一次学习使用Netty的时候，总是下意识的把这两个与 I/O的输入输出流联系在一起，但是随着使用的增多，如果，ChanneloutboundHandler代表输出流，为什么又有read方法？

正确理解

从Stackoverflow的这篇文章能够理解一二

Netty是由事件驱动的， 事件有 inbound 和 outbound 两种，分别由ChannelInboundHandler和ChannelOutboundHandler负责处理 。之间的区别便是

触发事件的源头。

inbound

触发事件的源头是外部，然后影响到应用程序内部。例如Netty的有关ChannelPipeline的JavaDoc中所写.

Inbound event propagation methods:

  • ChannelHandlerContext.fireChannelRegistered()
  • ChannelHandlerContext.fireChannelActive()
  • ChannelHandlerContext.fireChannelRead(Object)
  • ChannelHandlerContext.fireChannelReadComplete()
  • ChannelHandlerContext.fireExceptionCaught(Throwable)
  • ChannelHandlerContext.fireUserEventTriggered(Object)
  • ChannelHandlerContext.fireChannelWritabilityChanged()
  • ChannelHandlerContext.fireChannelInactive()
  • ChannelHandlerContext.fireChannelUnregistered()

outbound

触发事件的源头是内部，然后影响到外部。例如Netty的有关ChannelPipeline的JavaDoc中所写.

Outbound event propagation methods:

  • ChannelOutboundInvoker.bind(SocketAddress, ChannelPromise)
  • ChannelOutboundInvoker.connect(SocketAddress, SocketAddress, ChannelPromise)
  • ChannelOutboundInvoker.write(Object, ChannelPromise)
  • ChannelHandlerContext.flush()
  • ChannelHandlerContext.read()
  • ChannelOutboundInvoker.disconnect(ChannelPromise)
  • ChannelOutboundInvoker.close(ChannelPromise)
  • ChannelOutboundInvoker.deregister(ChannelPromise)

ChannelGroup

消息广播

在Netty中提供了ChannelGroup接口，是线程安全的，该接口继承Set接口，可以通过ChannelGroup可管理服务器端所有的连接的Channel，然后对所有的连接Channel广播消息。 并且ChannelGroup可以自动remove在ChannelGroup中已经关闭的Channel。因此也不用分心来关注Channel的生命周期。

Channel的生命周期：

  • ChannelRegistered
  • ChannelActive
  • ChannelInactive
  • ChannelUnregistered

简化关闭

如果ServerChannel和非ServerChannel都存在于同一个ChannelGroup中，则首先对此组上的ServerChannel执行所有请求的I / O操作，然后对其他Channel执行操作。

ChannelGroupFuture close()

Closes all Channels in this group. If the Channel is connected to a remote peer or bound to a local address, it is automatically disconnected and unbound.

  • Returns:

    the ChannelGroupFuture instance that notifies when the operation is done for all channels
