理解Spring中的@Autowired, @Resource and @Inject注解含义(译)
spring
spring
发布日期:   2020-01-05
更新日期:   2020-01-06
文章字数:   1.8k
阅读时长:   8 分
阅读次数:  

总览

常用的关于依赖注入( dependency injection)的相关注解有 @Resource @Autowired @Inject 仨个注解。 这些注解替我们为类 提供了一种声明式的方法来解析依赖项。

例如：

@Autowired
    public Helloservice service

代替了下面的例子，实现了相同的功能

Helloservice service = new HelloserviceImpl();

三个注解中的两个是属于 Java extension package 的注解，分别是 javax.annotation.Resource 和 javax.inject.Inject .而 @Autowired 注解属于

org.springframework.beans.factory.annotation

这三个注解都可以通过字段注入或者setter注入来解析依赖项

@Resource 注解

@Resource注解是属于JSR-250 规范请求中的一部分，这个注解有如下的匹配选项。

根据优先级排列：

  • Match by Name

  • Match by Type

  • Match by Qualifier

    这三种匹配方式同时适用于setter和字段的注入。

字段注入

Match by Name

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestResourceNameType.class)
public class FieldResourceInjectionIntegrationTest {

    //通过bean的名称作为属性值传给@Resource注解实现了通过名称解析依赖项
    @Resource(name="namedFile")
    private File defaultFile;

    @Test
    public void givenResourceAnnotation_WhenOnField_ThenDependencyValid(){
        assertNotNull(defaultFile);
        assertEquals("namedFile.txt", defaultFile.getName());
    }
}

@Configuration
public class ApplicationContextTestResourceNameType {

    //bean的名称,如果不指定自定义名称，则Bean名称将默认为方法名称
    @Bean(name="namedFile")
    public File namedFile() {
        File namedFile = new File("namedFile.txt");
        return namedFile;
    }
}

如果相应的bean在Application Context 中没有定义，将会抛出org.springframework.beans.factory.NoSuchBeanDefinitionException 异常

Match by Type

与 Match by Name 不同的是，只需要将@Resource 那一行去掉name属性值即可

@Resource
private File defaultFile;

基于类型的查找 这代表的就是spring要查找与设置的属性类型相同的bean，如果这种类型的bean不止一个，就会抛出异常

Match by Qualifier

这一次在ApplicationContextTestResourceQualifier中定义两个bean

@Configuration
public class ApplicationContextTestResourceQualifier {

    @Bean(name="defaultFile")
    public File defaultFile() {
        File defaultFile = new File("defaultFile.txt");
        return defaultFile;
    }

    @Bean(name="namedFile")
    public File namedFile() {
        File namedFile = new File("namedFile.txt");
        return namedFile;
    }
}

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestResourceQualifier.class)
public class QualifierResourceInjectionIntegrationTest {

    @Resource
    private File dependency1;

    @Resource
    private File dependency2;

    @Test
    public void givenResourceAnnotation_WhenField_ThenDependency1Valid(){
        assertNotNull(dependency1);
        assertEquals("defaultFile.txt", dependency1.getName());
    }

    @Test
    public void givenResourceQualifier_WhenField_ThenDependency2Valid(){
        assertNotNull(dependency2);
        assertEquals("namedFile.txt", dependency2.getName());
    }
}

直接运行则会抛出异常org.springframework.beans.factory.NoUniqueBeanDefinitionException,这可以联想上面的 By Type 示例，这表示了，没有特殊的标识，spring在这两个相同的类型的bean不知道该选择哪个，解决这个问题，可以在@Resource注解下面增加 @Qualifier 注解。

@Resource
@Qualifier("defaultFile")
private File dependency1;

@Resource
@Qualifier("namedFile")
private File dependency2;

如果在Application Context中定义了多个bean，使用@Qualifier 通过特定的标识可以有效的消除混淆。

基于Setter 的注入

Match by Name

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestResourceNameType.class)
public class MethodResourceInjectionIntegrationTest {

    private File defaultFile;
     //setter注入解析依赖项，是通过注解引用变量的相应的setter方法完成的。
    @Resource(name="namedFile")
    protected void setDefaultFile(File defaultFile) {
        this.defaultFile = defaultFile;
    }

    @Test
    public void givenResourceAnnotation_WhenSetter_ThenDependencyValid(){
        assertNotNull(defaultFile);
        assertEquals("namedFile.txt", defaultFile.getName());
    }
}

bean的命名必须和属性值严格对应，和如果传给name的值没有相应的bean命名，便会抛出异常NoSuchBeanDefinitionException

Match by Type

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestResourceNameType.class)
public class MethodByTypeResourceIntegrationTest {

    private File defaultFile;

    @Resource
    protected void setDefaultFile(File defaultFile) {
        this.defaultFile = defaultFile;
    }

    @Test
    public void givenResourceAnnotation_WhenSetter_ThenValidDependency(){
        assertNotNull(defaultFile);
        assertEquals("namedFile.txt", defaultFile.getName());
    }
}

程序依旧运行正常，如果我们把defaultFile 的类型改成其他的类型，例如String，程序便会抛出NoSuchBeanDefinitionException 异常。

Match by Type

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestResourceQualifier.class)
public class MethodByQualifierResourceIntegrationTest {

    private File arbDependency;
    private File anotherArbDependency;

    @Test
    public void givenResourceQualifier_WhenSetter_ThenValidDependencies(){
      assertNotNull(arbDependency);
        assertEquals("namedFile.txt", arbDependency.getName());
        assertNotNull(anotherArbDependency);
        assertEquals("defaultFile.txt", anotherArbDependency.getName());
    }

    @Resource
    @Qualifier("namedFile")
    public void setArbDependency(File arbDependency) {
        this.arbDependency = arbDependency;
    }

    @Resource
    @Qualifier("defaultFile")
    public void setAnotherArbDependency(File anotherArbDependency) {
        this.anotherArbDependency = anotherArbDependency;
    }
}

和字段注入的示例类似，主要是对于定义有多个bean时，使用@Qualifier来进行特定的标识。

@Inject 注解

@Inject注解属于JSR-330规范请求，根据优先级排列执行方式 有如下顺序。

  • Match by Type
  • Match by Qualifier
  • Match by Name

如果我们要使用@Inject注解，我们需要添加相应依赖。

Gradle:

testCompile group: 'javax.inject', name: 'javax.inject', version: '1'


Maven:
<dependency>
    <groupId>javax.inject</groupId>
    <artifactId>javax.inject</artifactId>
    <version>1</version>
</dependency>

字段注入

Match by Type

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestInjectType.class)
public class FieldInjectIntegrationTest {

    @Inject
    private ArbitraryDependency fieldInjectDependency;

    @Test
    public void givenInjectAnnotation_WhenOnField_ThenValidDependency(){
        assertNotNull(fieldInjectDependency);
        assertEquals("Arbitrary Dependency",
          fieldInjectDependency.toString());
    }
}

与@Resource不同的是，@Inject的执行优先级最高的是 Match by Type，所以，即使我们引用的变量名与bean的名字不同，@Inject也可以成功解析依赖。

Match by Qualifier

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestInjectQualifier.class)
public class FieldQualifierInjectIntegrationTest {

@Inject
@Qualifier("defaultFile")
private ArbitraryDependency defaultDependency;

@Inject
@Qualifier("namedFile")
private ArbitraryDependency namedDependency;

    @Test
    public void givenInjectQualifier_WhenOnField_ThenDefaultFileValid(){
        assertNotNull(defaultDependency);
        assertEquals("Arbitrary Dependency",
          defaultDependency.toString());
    }

    @Test
    public void givenInjectQualifier_WhenOnField_ThenNamedFileValid(){
        assertNotNull(defaultDependency);
        assertEquals("Another Arbitrary Dependency",
          namedDependency.toString());
    }
}

​ 在使用@Qualifier注解之后，spring将会寻找严格对应@Qualifier注解的属性值的bean。反之，将会抛出异常。

Match by Name

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestInjectName.class)
public class FieldByNameInjectIntegrationTest {

    @Inject
    @Named("yetAnotherFieldInjectDependency")
    private ArbitraryDependency yetAnotherFieldInjectDependency;

    @Test
    public void givenInjectQualifier_WhenSetOnField_ThenDependencyValid(){
        assertNotNull(yetAnotherFieldInjectDependency);
        assertEquals("Yet Another Arbitrary Dependency",
          yetAnotherFieldInjectDependency.toString());
    }
}

​ 和@Resource一样，将会寻找和name属性值相同命名的bean，如果找不到，则抛出NoSuchBeanDefinitionException异常。

基于Setter 的注入

基于Setter 的注入，和@Resource是相同的，都是在setter方法上在文章

@Autowired 注解

@Autowired注释的行为类似于@Inject注释。唯一不同的是@Autowired注解是Spring框架的一部分。该注释具有与@Inject注释相同的执行方法，按优先级排列

  • Match by Type
  • Match by Qualifier
  • Match by Name

字段注入

Match by Type

@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestAutowiredType.class)
public class FieldAutowiredIntegrationTest {

    @Autowired
    private ArbitraryDependency fieldDependency;

    @Test
    public void givenAutowired_WhenSetOnField_ThenDependencyResolved() {
        assertNotNull(fieldDependency);
        assertEquals("Arbitrary Dependency", fieldDependency.toString());
    }
}
@Configuration
public class ApplicationContextTestAutowiredType {

    @Bean
    public ArbitraryDependency autowiredFieldDependency() {
        ArbitraryDependency autowiredFieldDependency =
          new ArbitraryDependency();
        return autowiredFieldDependency;
    }
}

与@Inject注解类似， 默认是以类型查找的，bean的命名和被@Autowired注解类名字不同并不影响。如果修改两者的类型不一致，则会抛出NoSuchBeanDefinitionException异常。

Match by Qualifier

@Configuration
public class ApplicationContextTestAutowiredQualifier {

    @Bean
    public ArbitraryDependency autowiredFieldDependency() {
        ArbitraryDependency autowiredFieldDependency =
          new ArbitraryDependency();
        return autowiredFieldDependency;
    }

    @Bean
    public ArbitraryDependency anotherAutowiredFieldDependency() {
        ArbitraryDependency anotherAutowiredFieldDependency =
          new AnotherArbitraryDependency();
        return anotherAutowiredFieldDependency;
    }
}
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestAutowiredQualifier.class)
public class FieldQualifierAutowiredIntegrationTest {

@Autowired
@Qualifier("autowiredFieldDependency")
private FieldDependency fieldDependency1;

@Autowired
@Qualifier("anotherAutowiredFieldDependency")
private FieldDependency fieldDependency2;

    @Test
    public void givenAutowiredQualifier_WhenOnField_ThenDep1Valid(){
        assertNotNull(fieldDependency1);
        assertEquals("Arbitrary Dependency", fieldDependency1.toString());
    }

    @Test
    public void givenAutowiredQualifier_WhenOnField_ThenDep2Valid(){
        assertNotNull(fieldDependency2);
        assertEquals("Another Arbitrary Dependency",
          fieldDependency2.toString());
    }
}

关于为什么这么写的原因已经不用再过多重复了。这三个注解理念大部分还是一致的。

Match by Name

当使用@Autowired批注注入字段依赖项时，必须将@ComponentScan注解与Application Context一起使用. ComponentScan 需要指明spring要去扫描的包位置。@ComponentScan默认扫描启动类的所在的包和子包下面所有的注解

//这个注解的意思是告诉spring框架ArbitraryDependency类是一个名为autowiredFieldDependency的组件
@Component(value="autowiredFieldDependency")
public class ArbitraryDependency {

    private final String label = "Arbitrary Dependency";

    public String toString() {
        return label;
    }
}
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(
  loader=AnnotationConfigContextLoader.class,
  classes=ApplicationContextTestAutowiredName.class)
public class FieldAutowiredNameIntegrationTest {

    @Autowired
    //@Autowired没有属性值，所以变量的命名必须和注册的组件名一致。反之抛出异常NoUniqueBeanDefinitionException
    private ArbitraryDependency autowiredFieldDependency;

    @Test
    public void givenAutowiredAnnotation_WhenOnField_ThenDepValid(){
        assertNotNull(autowiredFieldDependency);
        assertEquals("Arbitrary Dependency",
          autowiredFieldDependency.toString());
    }
}

基于Setter 的注入

@Autowired注解的setter注入类似于@Resource setter注入。注解相应的setter。基于字段的依赖项注入所遵循的执行方法也适用于基于setter的注入。
