AOP面向切面编程之学习理解

AOP做到了什么？

OOP (面向对象编程) 更多的操作还是在类的纵向方向上，例如类的继承，类的实现。这就导致了一种情况，例如我们需要横向方向上不同模块的服务，像增加 处理异常并打日志，安全校验，权限管理等等新功能，就不可避免的要在每一个类的方法上写 “新功能” 的代码。这就会出现两个问题。

  • 代码冗杂： 不同模块不同功能的代码冗杂在 一个方法里。
  • 代码重复： 需要在每一个方法里都要写一遍“新功能”的代码。

因此， Aspect-OrientedProgramming，面向切面编程。AOP是对OOP（Object-Oriented Programing）面向对象编程的一种很好的补充和完善。 AOP就是为了解决这种问题而来的。

java中的动态代理

AOP中常见的术语

  • 切面（Aspect）： 一个关注点的模块化，AOP的核心就是切面，它将多个类的通用行为封装为可重用的模块。该模块含有一组API提供 cross-cutting功能。在Spring AOP中，切面通过带有@Aspect注解的类实现。
  • 横切面拦截点（ Cross-cutting concerns ）：这是一种概念，例如我们想要在执行方法前都要进行权限校验，那这个校验就是Cross-cutting concerns。在切面中定义了对横切面拦截点的方法。
  • 织入（weave）： 将切面与建议对象链接的过程。 它可以在加载时，编译时或在运行时完成。 Spring AOP在运行时进行织入。
  • 连接点 （ Join point ）：连接点代表应用程序插入AOP切面的地点。
  • 切入点（Pointcut）：切入点是一个或者一组连接点，建议（Advice）将会在这些位置执行，切入点支持使用 execution表达式来匹配多个连接点。
  • 建议（Advice）：将会在连接点上执行的动作，来完成我们需要的功能—日志处理，权限检查等等。建议（Advice）有五种类型
    • Before：在方法执行前的前置操作
    • After：在方法执行后的后续操作，无论正常返回抑或是异常退出。
    • Around：环绕，在方法调用前后的触发的操作。
    • AfterReturning： 在方法完成后执行操作，必须是正常返回，没有任何异常。
    • AfterThrowing ： 在方法引发运行时异常后执行操作。

代码示例

下面的代码是进行主从数据库读写分离的切面类的代码，以此来作为示例。

package me.superning.userpassbook.bean;

import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.springframework.stereotype.Component;

/**
 * @author superning
 * @Classname DataSourceAop
 * @Description 数据库读写分离的切面类
 * @Date 2020/2/15 16:20
 * @Created by superning
 */
@Aspect
@Component
public class DataSourceAop {

    @Pointcut("!@annotation(me.superning.userpassbook.annotation.Master) " +
            "&& (execution(* me.superning.userpassbook.service..*.select*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.find*(..)))")
    public void readPointcut() {

    }


    @Pointcut("@annotation(me.superning.userpassbook.annotation.Master) " +
            "|| execution(* me.superning.userpassbook.service..*.insert*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.add*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.update*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.edit*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.delete*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.remove*(..))")
    public void writePointcut() {

    }

    @Before("writePointcut()")
    public void write(JoinPoint joinPoint){
        DBcontextHolder.master();
    }
    @Before("readPointcut()")
    public void read(){
        DBcontextHolder.slave();
    }


    }

首先利用@Aspect注解，声明这个类为切面类，然后使用@Component注解，将这个类作为组件注册进Spring容器里。

@Component
public class DataSourceAop {
    .............
}

然后，我使用@Pointcut注解，创建了两个切入点，readPointcut切入点使用表达式在符合（使用了自定义的Master注解，以及服务Service是查询服务时）的条件下执行后续建议（Advice）。writePointcut切入点与readPointcut切入点目的类似，只不过表达式的条件不同。

            "&& (execution(* me.superning.userpassbook.service..*.select*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.find*(..)))")
    public void readPointcut() {

    }


    @Pointcut("@annotation(me.superning.userpassbook.annotation.Master) " +
            "|| execution(* me.superning.userpassbook.service..*.insert*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.add*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.update*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.edit*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.delete*(..)) " +
            "|| execution(* me.superning.userpassbook.service..*.remove*(..))")
    public void writePointcut() {

    }

针对不同的切入点，关联使用了不同的建议（Advice），当某切入点条件符合之后，根据我们定义的Advice的不同，将会连接点对应的位置上执行我们定义的Advice方法。

    @Before("writePointcut()")
    public void write(JoinPoint joinPoint){
        DBcontextHolder.master();
    }
    @Before("readPointcut()")
    public void read(JoinPoint joinPoint){
        DBcontextHolder.slave();
    }
