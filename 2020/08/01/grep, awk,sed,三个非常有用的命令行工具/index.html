知识记录

由于最近在实习中需要经常编写shell脚本，其中不免要大量接触到 awk 、grep、sed 这三个重要的命令，本文就这三条命令做一个简单的知识记录。

grep = global regular expression print

grep 通常会处理输入的文件信息，并且输出符合匹配的行或者信息。grep会从第一行开始，一行一行的一直到最后一行，grep会将每一行拷贝一份到buffer缓冲区中，并且与我们输入的匹配信息进行比较，如果匹配的话，就将这一行打印到屏幕上。

grep 不会对信息中的数据进行储存，改变。只是简单的对行进行搜索。

接下来利用下面的这个例子，例子命名为file ，来讲解grep的参数用法。

boot
book
booze
machine
boots
bungie
bark
aardvark
broken$tuff
robots
grep -n "boo" file

1:boot
2:book
3:booze
5:boots

-n 参数，会让grep输出匹配的行前面加上这一行的行数。

grep -v "boo" file

machine
bungie
bark
aardvark
broken$tuff
robots

-v 参数， 会让grep输出不匹要求的行。

grep -c "boo" file

4

-c 参数，会让grep输出匹配要求的行的数量。

grep -l "boo"  *

file

-l 参数，会让grep 输出匹配要求的文件名字

grep -i "BOO" file

boot
book
booze
boots

-i 参数， 会让grep 不在分辨大小写的区别。

grep -x "boo" file

-x 参数，会让grep输出精准匹配的行，因此，上面的输出为空。

正则表达式

如果想要更广泛的正则表达式命令，那么就使用“grep -E”(也称为
如egrep命令)。例如 ？将匹配1或0次出现的前一个字符:

grep -E "boots?" file

boot
boots
grep -E "boot|boots" file

boot
boots

如果想要了解更多的正则 ，—> https://gnosis.cx/publish/programming/regular_expressions.html

AWK

awk 是一个文本扫描与处理的语言，由Aho, Weinberger & Kernighan 创造，awk可以非常的复杂，在这个作为知识笔记的博文全写出来肯定不现实。但是呢，awk也有一些比较简单的用法，如果需要更多更复杂的用法，网上也有很多的资料，如果你擅长Linux系统，你也可以用 man awk 命令

基础

首先，awk命令通常有一个模板，如果我们要使用awk命令，最好对照着模板来编写。

BEGIN { …. initialization awk commands …}
{ …. awk commands for each line of the file…}
END { …. finalization awk commands …}

上面也介绍的很明白了，awk分为三部分，先是BEGIN的初始awk命令，然后是awk对文件操作的命令，最后是END中 结束命令。awk也像grep一样，支持正则表达式。另外awk还支持复杂的数学运算，也支持关联数组。

在awk中，awk会将每一行分为几个部分来对待，默认情况下，awk是使用空格来分割不同的部分，当然，你也可以自定义需要的分隔符号。

this is a awk text demo

在默认情况下，上面的文本会被分割成6个部分，$1 是 this, $ 6是demo。 $0 就是指的 这一行。

简单的例子

ning@LXN:~/got$ ll
total 16
drwxr-xr-x  2 ning ning 4096 Aug  1 21:16 ./
drwxr-xr-x 11 ning ning 4096 Aug  1 16:42 ../
-rw-r--r--  1 ning ning   70 Aug  1 16:47 a_file
-rw-r--r--  1 ning ning  670 Aug  1 21:16 test
PS: 上面是我本机上的Linux在got文件夹的ll命令的输出。有a_file和test两个文件。
如果我们想要知道在got文件夹下的所有文件的总大小是多大，利用awk怎么做到呢？

首先看一下，我们想要的内容在第三行之后开始出现，然后文件的大小内容在每一行的第5列。这时有了两个关键信息，有效的行的行数要比3大，如果按照默认来说，有效的列数应该是第5列。这时我们便可以编写如下的语句。

ning@LXN:~/got$ ll | awk 'BEGIN{sum=0} {if(NR>3) {sum=sum+$5}} END {print "总大小"sum}'

总大小740

上面有几个需要注意的点 ，最后的END如果忘记加上，那么print的这一条命令就会对每一行执行。因此，如果想要命令在最后结束的时候执行，想要加上END。另外就是注意 awk 后面的命令应该用单引号包起来。

关于awk的更多用法可以参考其他的资料，也可以在需要的使用根据需求去查询相关的操作。

sed = stream editor

sed对中的输入流(文件或管道输入)执行基本的文本转换，它是非常高效的，sed可以在命令行中使用，也可以在shell脚本中使用，以非交互方式编辑文件。最有用的功能是将一个字符串“搜索并替换”到另一个字符串。

可以使用‘-e’选项将sed命令嵌入到调用sed的命令行中，或者。
将它们放在单独的文件中，例如‘sed.in’，然后使用‘-f sed.in’选项调用sed。大多数情况下，sed命令很复杂并且涉及大量正则表达式。

sed -e 's/input/output/' my_file

将从my_file回显每一行到标准输出，更改每一行上第一次出现的“输入” 进入“输出”行。注意sed是面向行的，所以如果你想改变每一行中的每个事件， 然后你需要把它变成一个贪婪搜索。

sed -e 's/input/output/g' my_file
