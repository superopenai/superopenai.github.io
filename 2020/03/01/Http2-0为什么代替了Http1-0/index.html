Http2.0为什么代替了Http1.0?

什么是HTTP？

简单来说，HTTP是超文本传输协议的英文简称，Http是基于CS 架构模型，客户端（服务接收方）和服务端（服务提供商）进行请求和响应的通信。

现在来说，Http2.0已经基本完全代替了Http1.0，拿Chrome这种主流浏览器来说，几乎所有主流浏览器都支持了Http2.0协议。但是为什么Http2.0最终代替了Http1.0呢？且听我慢慢道来。

Http1.x时代

Http/1.1 由Timothy Berners-Lee于1989年开发，作为万维网的通信标准 ，Http协议是应用层协议，该协议用于客户端与远程服务提供服务器进行交互。

通常客户端会使用 GET/POST 协议向服务端发送请求，而服务端收到请求后会把请求的资源（例如HTML页面）响应给客户端。

传输模型—-管道模型和行头阻塞

我们打开一个页面，我们需要的是最后渲染完成的整体页面，但是一个完整的页面同通常是包含很多资源的，HTML页面，图片资源，CSS样式等等… 但是在Http1.0时代，每个资源都对应着一次TCP连接。举个例子就是，图片A需要一个TCP连接，图片B需要一个TCP连接，CSS样式也需要一个TCP连接。建立的TCP连接的数量和这个页面上的资源成正比，这种行为无疑是效率十分低下的，浪费了大量的TCP连接资源，页面的载入速度也很慢。

在Http1.1时代，为了解决1.0时代为每一个资源都建立一个TCP的问题，Http1.1带来管道模型，所谓管道模型，就是放弃1.0时的TCP短连接，引入TCP长连接，在这一次TCP长连接时，客户端可以依次发送对多个资源的多个请求，服务端也响应的进行多次资源响应。

看似1.1带来的管道模型解决1.0的问题，看来已经很完美了，但是1.1的管道模型有一个天然缺点，由于多个数据包在到达同一目的地时无法相互传递，因此存在队列头部的请求无法检索其所需资源的情况，该请求将阻塞其后面的所有请求。

这称为行头(HOL)阻塞，这是在HTTP/1.1中优化连接效率的一个重大问题。

Http2.0时代

2015年，IETF推出了Http2.0协议，也是Http协议的第二个主要版本，Http2.0主要是基于谷歌在早期推出的SPDY协议（已经废弃）的基础上研发的。

Http2.0 的改进之处：

  • 与Http1.1的高级兼容
  • 协议协商机制-协议选举
  • 加快页面的载入速度：
    • 压缩请求头
    • 使用二进制对请求/响应进行编码，提高传输的灵活性
    • 服务端资源PUSH
    • 单次TCP连接的多路复用

接下来就以Http2.0是怎么加快页面的加载速度的，来给大家讲解一下这4处改进

TCP连接的多路复用

HTTP/2 能够在单次的TCP连接中并行发送多次请求，这是HTTP/2 最优越的特性，这使得我们能够异步的从服务端下载我们所需要的资源，这有效的减少了额外的往返时间（RTT），使得页面的加载速度变得更快，即使我们没有使用任何的优化策略。

报头压缩

HTTP/2 压缩了报头大量的冗余信息，使用HPACK规范来安全简单的压缩报头，在客户端和服务器端都为维护这一份之前请求过的报头列表。

HPACK 会在发送到服务器端之前对每一个值进行压缩，当压缩后的报头传输到服务端后，服务端会根据其维护的报头列表来重建完成的报头信息。

二进制编码

在HTTP1.x时代，传输的信息是文本信息，在HTTP/2，将传输文本信息改进为把请求信息二进制编码后再传输，有效的简化了复杂性。并且还有如下优点：

  • 更低的传输资源开销
  • 更少出错
  • 有效利用网络资源
  • 消除HTTP1.x值容易被修改的问题
  • 可以启用HTTP/2的其他功能，压缩，多路复用，TLS小绿🔒
  • 降低了网络延迟并提高了吞吐量。

服务端资源PUSH

这项功能使得服务端在收到客户端的请求后，除了响应请求的资源之外，还会向客户端推送其他的资源。举个例子，如果我们请求X资源，服务端收到请求后，发现X资源引用着 Y和Z 资源，服务端认为你以后还会请求Y/Z资源，然后服务端就一次性的响应给客户端请求的X资源和额外的Y和Z资源。

这就有如下好处：

  • 客户端把额外推送的资源放在缓存中，加快页面渲染速度
  • 服务端可以在单次TCP连接下，多路复用的向客户端push资源
  • 服务端可以根据资源的优先级来推送
  • 客户端可以限制服务端Push过来的资源的数量

速度

同样请求100张图片，HTTP1.1和HTTP2.0差距明显。

HTTP1.1—12.97s (连续载入三次)

HTTP2.0—3.17s (连续载入三次)

总结

HTTP 2.0从各种程度上都超越了1.x，既解决了1.x的问题还更快的加载渲染页面，现在主流的浏览器基本都支持了HTTP2.0。
